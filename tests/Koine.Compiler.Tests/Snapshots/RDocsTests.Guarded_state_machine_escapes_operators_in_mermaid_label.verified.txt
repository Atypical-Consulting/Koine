# Ordering

## Aggregates

### Order

**Root entity:** `Order` (identified by `OrderId`)

```mermaid
classDiagram
    class Order {
        <<aggregate root>>
        +OrderId id
        +Decimal total
        +OrderStatus status
        +submit()
    }
```

#### Lifecycle

```mermaid
stateDiagram-v2
    [*] --> Draft
    Draft --> Submitted: when total &gt; 0 &amp;&amp; total &lt;= 100
    Submitted --> Done
    Done --> [*]
```

| From | To | Guard |
| --- | --- | --- |
| `Draft` | `Submitted` | `total &gt; 0 &amp;&amp; total &lt;= 100` |
| `Submitted` | `Done` |  |
| `Done` | _(terminal)_ | |

#### Commands

##### `submit()`

**Preconditions:**
- only a draft can be submitted

**Effects:**
- `status -> Submitted`

## Domain Types

### OrderStatus — enum

Values: Draft, Submitted, Done
