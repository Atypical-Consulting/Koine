# Repo-profile schema

The shape of `.claude/skills/repo-profile.md`. `get-repo-profile` fills this from detection (Step 3 of
the skill) and writes it; `create-issue` / `implement-issue` / `merge-pr` read it at their preconditions
step. Keep **every section** — leave `<!-- TODO -->` markers for anything undetectable rather than
dropping the field. Values are consumed *verbatim* (label strings, commit identity, commands), so record
them exactly as the repo uses them.

## Worked example — Koine (`.claude/skills/repo-profile.md`)

What detection produces for this repo, as a reference for the field values' grain:

````markdown
# Repo profile

## Identity
- **Repo:** Atypical-Consulting/Koine
- **Default branch:** main

## Commit identity
- **Author line:** `git -c user.email=phmatray@gmail.com -c user.name="Philippe Matray"`
- **Source:** CLAUDE.md commit-identity rule (use the GitHub identity, not the work email)

## Build & test
- **Build:** `dotnet build`
- **Full test:** `dotnet test` (needs the `wasm-tools wasm-experimental` workloads — CI-only here)
- **Single-suite filter:** `dotnet test --filter "FullyQualifiedName~<Suite>"`
- **Format/lint apply:** `dotnet format Koine.slnx`
- **Format/lint verify (gate):** `dotnet format Koine.slnx --verify-no-changes --no-restore`
- **Prerequisites / caveats:** wasm workloads can't be installed locally — target individual projects;
  `IDE1006` naming can't be auto-fixed (rename by hand). Studio FE tests: `npx vitest run --project '!storybook'`.

## CI gates
- `dotnet build`
- `dotnet test`
- `dotnet format Koine.slnx --verify-no-changes --no-restore`

## Integration style
- **Merge mode:** squash
- **PR title convention:** Conventional Commits prefix `<type>(scope):` required (CI `pr-title-lint.yml` /
  `semantic-pull-request`; feeds `release-please`, ADR 0002) — derive from issue label (`bug`→`fix`,
  `enhancement`→`feat`), don't reuse the issue title verbatim; title also ends in `(#issue)`; squash
  appends `(#PR)` → `<type>(scope): subject (#issue) (#PR)`
- **Branch naming:** `feat/<issue>-<slug>`

## Labels
- **Type:** `bug`, `enhancement`
- **Priority tiers:** `priority: high` (Tier 1 — core deliverable) · `priority: medium` (Tier 2 — expected, partial/absent) · `priority: low` (Tier 3 — polish)
- **Effort sizes:** `effort: S` (~hours–1d) · `effort: M` (~few days) · `effort: L` (~1–2wk / cross-layer) · `effort: XL` (multi-week/phased)
- **Scope/area:** `studio` — when the work concerns the Koine Studio IDE (`tooling/koine-studio`); finer scope via the issue-template Area dropdown

## Issue templates
- **Location:** `.github/ISSUE_TEMPLATE/`
- **Forms:** feature_request.yml, bug_report.yml, config.yml
- **Default for ideas:** feature_request · **for defects:** bug_report

## Conflict hot-spots
| File | Why it collides | Resolution |
|------|-----------------|------------|
| `Directory.Build.props` (`<Version>`+`<InformationalVersion>`) | every feature bumps the patch version | take the **higher** (both lines match) |
| `CHANGELOG.md` | everyone appends | **union** |
| `README.md`, `USER-STORIES.md`, feature catalogue, `website/` | parallel features document themselves | **union** |
| Verify snapshots `*.verified.txt` | two branches change emitted output | **regenerate** (re-run the test, accept `.received.txt`) |
| `tooling/koine-studio/package-lock.json` | lockfile churn | **regenerate** (`npm install`) |
| Emitter partials, `UsingCollector`, `CSharpNaming` | both add methods/usings | **union** |
| `R##…Tests.cs`, focused suites | both append tests | **union** |
| `.csproj`, `Koine.slnx` | both add files/projects | **union** item groups |
| `tooling/koine-studio/src/**` | parallel Studio features | **union** additive; same component edited both ways → understand both intents |

## Architecture grain / invariants
- Keep `Ast/` target-agnostic — never leak a C# concept into the shared model.
- Touch layers in order: grammar (`.g4`) → builder visitor (`Parsing/`) → semantic model (`Ast/`) → validators (`Semantics/`) → emitter (`Emit/<Target>/`) → tests.
- Do NOT set `TreatWarningsAsErrors` — generated demo code may warn and must still build.

## Worktree home
- `.claude/worktrees/` (git-ignored)

## Environment gotchas
- Raw `git fetch`/`push` can be sandbox-blocked while `gh` works (`port 443` timeout) — re-run those with the sandbox disabled; use `git -C <path>`, not `cd`.
- The wasm workloads are CI-only here; the full `dotnet test` over the solution won't restore locally — target individual projects.
````
