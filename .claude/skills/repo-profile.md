## Integration style
- **Merge mode:** squash
- **PR title convention:** two constraints, both CI-enforced by `pr-title-lint.yml`
  (`amannn/action-semantic-pull-request`). (1) **Conventional Commits prefix** —
  `<type>[(scope)]: subject`, type ∈ `feat|fix|docs|style|refactor|perf|test|build|ci|chore|revert`;
  the squash-merged title feeds `release-please` (ADR 0002), so a bare title is rejected *and* cuts no
  release. Issue titles are **not** conventional — derive the prefix (label `bug`→`fix`,
  `enhancement`→`feat`; scope like `emit-cs`/`emit-php`/`gbnf`/`wasm` from the touched area), don't pass
  the issue title through verbatim. (2) **`(#issue)` suffix** — the squash appends `(#PR)`, giving the
  canonical `<type>(scope): subject (#issue) (#PR)` shape on `main`.
- **Branch naming:** `feat/<issue>-<slug>`

## Conflict hot-spots (merge-with-main resolutions)
| File | Why it collides | Resolution |
|------|-----------------|------------|
| `Directory.Build.props` (`<Version>` **and** `<InformationalVersion>`) | every feature bumps the patch version | take the **higher** (both lines must match; never double-increment) |
| `CHANGELOG.md` | everyone appends an entry | **union** — keep both entries |
| `README.md`, `USER-STORIES.md`, feature catalogue, `website/` docs | parallel features document themselves | **union** — keep both sides' sections |
| Verify snapshots `*.verified.txt` | two branches changed emitted output | **regenerate** — take a side, re-run the test, accept `.received.txt` → `.verified.txt` |
| `tooling/koine-studio/package-lock.json` | lockfile churn | **regenerate** — take a side, `npm install` to rebuild |
| `tooling/koine-studio/package.json` | both add deps/scripts | **union** deps/scripts, then regenerate the lockfile |
| Emitter partials (`Emit/CSharp/CSharpEmitter.*.cs`), `UsingCollector`, `CSharpNaming` | both add methods/usings | **union** — let the build catch a real clash |
| Test files (`R##…Tests.cs`, focused suites) | both append tests | **union** — keep both sets |
| `.csproj`, `Koine.slnx` | both add files/projects | **union** the item groups |
| `tooling/koine-studio/src/**` | parallel Studio features touch shared components | **union** where additive; same component edited both ways → understand both intents |
- **Rule of thumb:** union additive files (docs, tests, changelog), regenerate derived files (snapshots,
  lockfiles), take-the-higher for the version; same-logic-both-sides is a real semantic conflict —
  understand both intents or stop and surface both sides.

## Architecture grain / invariants (shape implementation plans to these)
- Keep `Ast/` **target-agnostic** — never leak a C#-specific concept into the shared semantic model.
- Touch layers in order: grammar (`.g4`) → builder visitor (`Parsing/`) → semantic model (`Ast/`) →
  validators (`Semantics/`) → emitter (`Emit/<Target>/`) → tests (a new `R##…Tests.cs` with snapshot +
  Roslyn coverage). A new emitter target is a new `Emit/<Target>/`, never target concepts in `Ast/`.
- Do **not** set `TreatWarningsAsErrors` — generated demo code may emit warnings and must still build.

## Environment gotchas
- Raw `git fetch`/`push` can be sandbox-blocked while `gh` works (`Failed to connect … port 443`);
  re-run just those network commands with the sandbox disabled. Local git (merge/commit/worktree) needs
  no network. Use `git -C <path>`, not `cd <path> && …` (a `cd` in a compound command gets reset).

  locally — target individual projects.
