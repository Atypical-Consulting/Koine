# Reference artifact (generated, for reference)

`ordering.rs.txt` is **generated by the Koine CLI** and committed here as a stable, browsable
snapshot — the live output tree (`demo/rust/generated/`) is git-ignored and rebuilt on every
`run.sh`, so it never shows up in diffs. This folder gives reviewers something concrete to look at
without running the tool.

Unlike the TypeScript/Python/PHP emitters (which write one file per type), the Rust emitter
bundles an entire bounded context into a single module file, so one file is the natural "a couple
of representative emitted source files" snapshot here — `ordering.rs.txt` is the whole `Ordering`
context from [`templates/starters/ordering`](../../../templates/starters/ordering): `DomainError`,
`OrderStatus`, `OrderLine`, `OrderId`, `Order`, `OrderRepository`, and `ProductId`.

Regenerate it (or the full crate) from the repo root:

```bash
dotnet run --project src/Koine.Cli -- build templates/starters/ordering --target rust --out /tmp/ordering-rs
```

The `.rs.txt` suffix is on purpose: `demo/rust/run.sh` only ever builds `demo/rust/{src,generated}`
as a crate (via `Cargo.toml`'s `koine-domain = { path = "generated" }`), so this reference copy is
never picked up as a duplicate module.

## Constructor parity across targets

`Order::new`'s trailing `status: Option<OrderStatus>` parameter (defaulting to `Draft` when
omitted) now matches the same "optional trailing constructor parameter" shape the sibling
TypeScript/PHP/C# references already render for this construct — see `../../typescript/reference/Order.ts.txt`
and `../../php/reference/Order.php.txt`. Before #1380, the Rust emitter dropped this member from
`new(...)`'s signature entirely and always initialized it to the literal default inline, with no way
for a caller outside the crate to construct an `Order` whose `status` was anything but `Draft`.
