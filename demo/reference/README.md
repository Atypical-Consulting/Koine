# Reference artifacts (generated, for reference)

These files are **generated by the Koine CLI** and committed here as a stable,
browsable snapshot. The live output tree (`demo/Pizzeria.Domain/Generated/`) is
git-ignored and rebuilt on every `dotnet build`, so it never shows up in diffs;
this folder gives reviewers something concrete to look at without running the tool.

The demo generates from the **`templates/pizzeria`** model (the single validated source
of truth — see issue #101), so these artifacts mirror that template's emitted output.

Regenerate any of them from the repo root:

| File | What it is | Command |
| --- | --- | --- |
| `pizzeria.glossary.md` | Ubiquitous-language glossary (R4.2), grouped by context/type | `dotnet run --project src/Koine.Cli -- build templates/pizzeria --glossary demo/reference/pizzeria.glossary.md` |
| `koine-check.txt` | A literal `koine check` transcript (R15.2) flagging a breaking change | `dotnet run --project src/Koine.Cli -- check examples/versioning/v2 --baseline examples/versioning/v1` |
| `emitted-cs/Order.cs.txt` | The emitted `Ordering.Order` aggregate (factory, command, state machine, invariants, events, optimistic-concurrency token) | `dotnet run --project src/Koine.Cli -- build templates/pizzeria --target csharp --out demo/Pizzeria.Domain/Generated` |
| `emitted-cs/Address.cs.txt` | The `Delivery.Address` value object — shape validation (regex), a derived one-line form, and the anchoring "a delivery needs a street" invariant | same as above |
| `emitted-cs/IGatewayToPaymentTranslator.cs.txt` | The anti-corruption translator interface emitted from the `acl { … }` block (R14.2): the external `Gateway.GatewayResult` → our clean `Payment.PaymentReceipt` | same as above |

The `.cs.txt` files use a `.txt` suffix on purpose so the demo project does
**not** compile them as duplicate copies of the live `Generated/` types.
