# Reference artifacts (generated, for reference)

These files are **generated by the Koine CLI** and committed here as a stable,
browsable snapshot. The live output tree (`demo/Shop.Domain/Generated/`) is
git-ignored and rebuilt on every `dotnet build`, so it never shows up in diffs;
this folder gives reviewers something concrete to look at without running the tool.

Regenerate any of them from the repo root:

| File | What it is | Command |
| --- | --- | --- |
| `shop.glossary.md` | Ubiquitous-language glossary (R4.2), grouped by context/type | `dotnet run --project src/Koine.Cli -- build demo/Shop.Domain/Models --glossary demo/reference/shop.glossary.md` |
| `koine-check.txt` | A literal `koine check` transcript (R15.2) flagging a breaking change | `dotnet run --project src/Koine.Cli -- check examples/versioning/v2 --baseline examples/versioning/v1` |
| `emitted-cs/Order.cs.txt` | The emitted `Ordering.Order` aggregate (factory, command, state machine, invariants, events, optimistic-concurrency token) | `dotnet run --project src/Koine.Cli -- build demo/Shop.Domain/Models --target csharp --out demo/Shop.Domain/Generated` |
| `emitted-cs/Currency.cs.txt` | The shared-kernel `Currency` smart enum, emitted **once** into a kernel namespace (R14.2) | same as above |

The two `.cs.txt` files use a `.txt` suffix on purpose so the demo project does
**not** compile them as duplicate copies of the live `Generated/` types.
