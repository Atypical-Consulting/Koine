# Reference artifacts (generated, for reference)

These two files are **generated by the Koine CLI** and committed here as a stable, browsable
snapshot — the live output tree (`demo/python/generated/`) is git-ignored and rebuilt on every
`run.sh`, so it never shows up in diffs. This folder gives reviewers something concrete to look at
without running the tool.

Both mirror the `Ordering` context in [`templates/starters/ordering`](../../../templates/starters/ordering).
Regenerate them (or the full tree) from the repo root:

```bash
dotnet run --project src/Koine.Cli -- build templates/starters/ordering --target python --out /tmp/ordering-py
```

| File | What it is |
| --- | --- |
| `order.py.txt` | The `Ordering.Order` aggregate root — id, lines (a `tuple[OrderLine, ...]`), status, entity equality |
| `order_line.py.txt` | The `Ordering.OrderLine` value object — a derived `subtotal` `@property` (`unit_price * quantity`) |

The `.py.txt` suffix is on purpose: the demo's own `mypy` run only ever type-checks
`demo/python/{generated,main.py}`, so these reference copies are never picked up as duplicate
modules.
