## Cross-cutting invariants (EVERY phase preserves these)

1. **`Ast/` (incl. `Ast/Bound/`) stays target-agnostic.** No C#/TS rendering, naming, type-mapping, or
   message-presentation string enters `Ast/`. Grep each new `Ast/` file for a target token — must be
   none. (Commit 4's relocated `SourceText` renders **Koine** syntax only — the one borderline helper, a
   reviewer confirms it carries no target string.)
2. **Zero conflict with the shipped TypeScript emitter.** **No signature change** to `ModelIndex`,
   `TypeResolver`, `KoineType`, or anything under `Emit/TypeScript/`. No `ModelIndex` helper is added for
   the binder/lowerer (they compose existing queries only). Commit 4 touches `Emit/CSharp/` **additively**
   (new overload), never `Emit/TypeScript/`.
3. **Value-equality-record / reference-identity discipline.** Every dictionary keyed by a `KoineNode`
   (or any value-equality record: `Member`/`TypeDecl`/`EnumMember`/`SpecDecl`/`ValueObjectDecl`) uses
   `ReferenceEqualityComparer.Instance`. Two value-equal nodes must NOT collide. This is the single
   recurring hazard (Commit 1's `SyntaxGraph`, Commit 2's `VisitList`, Commit 3's `BindingTable`, Commit
   4's `BoundModel` cache) — each phase has a dedicated regression test that fails loudly if dropped.
4. **Snapshots stay green or are deliberately re-accepted.** Commits 2 and 3 are off the emit path —
   **zero** snapshot movement expected. Commit 4's contract is **zero `.received.txt` diffs**; any
   movement is a slice bug, not an accepted change. No snapshot is overwritten without review.
5. **Lazy, thread-safe `SemanticModel` artifacts.** New per-model artifacts (`_binding`, `_bound`) mirror
   Commit 1's `_graph`: `Lazy<…>(LazyThreadSafetyMode.ExecutionAndPublication)`, so the emit path forces
   only what a migrated slice needs and a shared/cached `SemanticModel` cannot race under concurrent LSP.

## Single-branch / single-PR strategy

- **One branch `feat/roslyn-arch-2-3-4`**, branched off `main`. **One clean commit per phase**, in
  dependency order (2 → 3 → 4). Each phase's commit message follows the repo convention
  (`feat(ast): …`), committed with the GitHub identity:
  `git -c user.email=phmatray@gmail.com -c user.name="Philippe Matray" commit -m "…"`.
- **`./build.sh` must be fully green (build + ~500 tests) before the next phase begins.** A phase that
  cannot go green is a hard stop — fix or revert that phase's commit before proceeding; never stack a
  phase on a red base.
- After Phase 4 is green: a **final adversarial review pass** (re-run the three critiques' must-fix
  checklists + `superpowers:requesting-code-review`), then **ONE PR** for all three commits. The PR
  description links the three specs and this plan and calls out the snapshot-zero-diff contract.
- **Rollback granularity:** because each phase is one self-contained commit, a regression discovered late
  can be reverted phase-by-phase (`git revert <phase-commit>`) without unwinding the others — except the
  hard dependency edges (reverting 2 forces reverting 3 and 4; reverting 3 forces reverting 4).

## Phase 1 — Commit 2: typed, source-generated visitor / rewriter

**Depends on:** Commit 1 (`SyntaxGraph`, merged) only.

**Scope.** Add a Roslyn incremental source generator that reads the `KoineNode` record hierarchy and
emits `KoineSyntaxVisitor` / `KoineSyntaxVisitor<TResult>` / `KoineSyntaxRewriter` /
`KoineSyntaxChildEnumerator` (with a generated dispatch switch — **no `Accept` member on nodes**).
Retarget the **one** production traversal (`SyntaxGraph.Build`: `NodeWalker.ChildNodes(node)` →
`ChildNodes.Of(node)`). Keep `NodeWalker` as the reflection oracle. Rewriter is shipped but has **no
production consumer** until Commit 4.

**Deliverables (files).**
- New `src/Koine.Compiler.SourceGen/Koine.Compiler.SourceGen.csproj` (`netstandard2.0`,
  `IsRoslynComponent`, `Microsoft.CodeAnalysis.CSharp` `PrivateAssets=all`, no net10-only BCL).
- New `src/Koine.Compiler.SourceGen/SyntaxVisitorGenerator.cs` (`IIncrementalGenerator`):
  discovers nodes/child-slots; **canonical child order** (inherited child slots base-first → own
  primary-ctor child params in source order → body `init` child props); **fails closed** (emits an
  error `Diagnostic` if zero concrete nodes found); `VisitList` uses `ReferenceEquals` per element;
  required-slot `Required(...)` guard throws on null; list-element-null throws.
- Modify `src/Koine.Compiler/Koine.Compiler.csproj` — analyzer `ProjectReference`
  (`OutputItemType="Analyzer" ReferenceOutputAssembly="false"`).
- Modify `src/Koine.Compiler/Ast/SyntaxGraph.cs` — **one line** (`Build` child enumeration).
- Modify `Koine.slnx` — add the generator project under `/src/`.
- New `tests/Koine.Compiler.Tests/SyntaxVisitorTests.cs`.
- `NodeWalker.cs` **unchanged** (retained as oracle).

**Verification gate (all must be green before Phase 2).**
- `./build.sh` green, **AND** the new `Koine.Compiler.SourceGen` project builds under it (explicit gate
  — generator coexists with `Directory.Build.props` + `netstandard2.0`).
- `SyntaxVisitorTests`: (1) **set-based** oracle equivalence vs `NodeWalker.ChildNodes` + a separate
  canonical-order assertion; (2) no-op rewriter returns `Assert.Same` root, every descendant identical;
  (3) one-leaf rewrite reallocates only the leaf→root spine (off-spine siblings `Assert.Same`);
  (4) `VisitList` value-equality trap — rewrite an element to a value-equal-but-different instance and
  assert the list+spine `Assert.NotSame`; (5) null/wrong-type contract (required→throw, optional→clear,
  list-element-null→throw, wrong-type→`InvalidCastException`); (6) **round-trip a parsed model through a
  no-op rewriter then re-run validate+emit** → byte-identical diagnostics + C#; (7) typed `Visit<int>`
  count == `NodeWalker.Descendants().OfType<…>().Count()`; (8) generator fail-closed (handles a
  known-present node).
- Existing `SyntaxGraphTests` stay green after the `Build` retarget (end-to-end equivalence oracle).

**Rollback note.** Fully self-contained: revert the single phase commit to remove the generator project,
the analyzer reference, the one `SyntaxGraph.cs` line, and the slnx entry. `NodeWalker` was never
removed, so `SyntaxGraph.Build` reverts to reflection with no other change. **Blocks 3 and 4** —
reverting this forces reverting them.

## Phase 2 — Commit 3: symbol-identity binder + reference-keyed binding table

**Depends on:** Commit 2 (the `Binder` subclasses the generated `KoineSyntaxVisitor`). *Fallback if 2
slips:* hand-rolled recursive descent reusing `ExprWalker` for the Expr body — but the plan lands 2
first, so the visitor dependency holds.

**Scope.** Introduce a `SymbolTable` (one interned `Symbol` per declaration, with an `init`-set
`ContainingSymbol` containment spine), a `Binder` (one walk via the Commit-2 visitor, scope stack,
records `node → interned Symbol`), and a reference-keyed `BindingTable` (BCL
`ReferenceEqualityComparer.Instance`). Expose `SemanticModel.GetSymbolInfo(node)` / `GetDeclaredSymbol(node)`
(never null — `ErrorSymbol.Instance` sentinel). Re-point `DefinitionAt`/`DeclaredSymbolAt` **and the
ad-hoc `new MemberSymbol`/`new TypeSymbol` sites** to return interned symbols (closes the
null-`ContainingSymbol` leak). **Byte-identical observable behavior** — no LSP/RefactorService caller
rewritten.

**Deliverables (files).**
- New `src/Koine.Compiler/Ast/SymbolTable.cs` — interned declaration symbols + containment over
  `ModelIndex`; build ordering **contexts → types → members/params/enum-members/id-VOs → locals/lambdas**.
- New `src/Koine.Compiler/Ast/Binder.cs` — `KoineSyntaxVisitor` subclass; resolution **reproduces**
  `GetSymbol(name, enclosingType)` / `ResolveReference` semantics (no new rules), then interns + records.
- New `src/Koine.Compiler/Ast/BindingTable.cs` — `Dictionary<KoineNode, Symbol>` (ref-keyed);
  `GetSymbolInfo`.
- Modify `src/Koine.Compiler/Ast/Symbols.cs` — add `Symbol? ContainingSymbol { get; init; }`; add
  `ContextSymbol` (NOT `NamespaceSymbol` — DSL vocabulary) / `ParameterSymbol` / `LocalSymbol` /
  `LambdaParameterSymbol` / `ErrorSymbol`; extend `SymbolKind` additively.
- Modify `src/Koine.Compiler/Ast/SemanticModel.cs` — lazy `_binding`; `GetSymbolInfo`/`GetDeclaredSymbol`;
  re-point `DefinitionAt`/`DeclaredSymbolAt`/`MemberOf`/`GetSymbol` construction sites to the table.
- New `tests/Koine.Compiler.Tests/BinderTests.cs`.
- **`SyntaxGraph.cs` NOT modified** (BCL comparer chosen, no `IdentityComparer` promotion).

**Verification gate (all must be green before Phase 3).**
- `./build.sh` green (~500 tests).
- `BinderTests`: (1) reference-key identity — two types each with `amount` field+ref → distinct
  `MemberSymbol`s (fails if `IdentityComparer` dropped); (2) interning — `GetDeclaredSymbol(decl)`
  `Assert.Same` on repeat; every `GetSymbolInfo(ref)` `Assert.Same` as `GetDeclaredSymbol`; N refs to one
  `Money` → one `TypeSymbol`; (3) equivalence oracle — every reference node matches legacy
  `DefinitionAt` on **`Kind` + `Name` + `DeclSpan`** (not DeclSpan alone) across the corpus, incl. a
  **convention-only `*Id`** (`product: ProductId`, no entity); (4) containment — every non-context symbol
  has non-null `ContainingSymbol`; two R13.2 `Money`s have distinct `ContextSymbol`s; (5) `ErrorSymbol`
  for unresolved, legacy shim still returns `null`.
- **Load-bearing oracle:** the full **R17 navigation + rename suite** (`InExpressionNavigationTests`,
  `NodeAtNavigationTests`, `KoineLanguageServiceTests`, `SemanticModelTests`) stays green after
  re-pointing — Verify/Roslyn meta-test are **blind** to the binder (off emit path), so this suite is the
  guard.

**Rollback note.** Off the emit path; the new `Ast/` files are additive and the `SemanticModel`
re-points are reversible. Revert the phase commit to restore fresh-alloc `GetSymbol` and remove the
table. **Blocks 4** — reverting this forces reverting 4.

## Phase 3 — Commit 4: bound-IR + Lowerer, proven by ONE VO-invariant-guard slice

**Depends on:** Commit 2 (`Lowerer : KoineSyntaxVisitor<BoundNode>`) **and** Commit 3
(`GetSymbolInfo` interned identity carried by `BoundReference`; `Assert.Same` tests require it). Not
implementable on today's tree.

**Scope.** Introduce a target-agnostic `BoundNode` record hierarchy in `Ast/Bound/` (Syntax
back-pointer; `KoineType Type` on every `BoundExpression`; Commit-3 `Symbol` on references) + a
`Lowerer` (Commit-2 visitor, reads resolution ONLY from `GetSymbolInfo`/`GetTypeInfo`, no new rules) +
`BoundModel` (caches lowered VO invariants, `ReferenceEqualityComparer` keyed by `ValueObjectDecl`).
Retarget **exactly one** emitter slice: **value-object invariant guards**, via a **new `BoundInvariant`
overload** of `WriteInvariantGuards`/`WriteGuard` (the raw-`Invariant` overload for entity/command
**stays**). Relocate `SourceText`/`SourceTextVisitor` **verbatim** into `Ast/Bound/` (C# message
default only — **TS untouched**). Translator still renders the condition from the **syntactic** `Expr`
(via `BoundInvariant.Syntax`) → **byte-identical C#**. `TypeScope.FromMembers(vo.Members)` mirrors the
emitter's scope.

**Deliverables (files).**
- New `src/Koine.Compiler/Ast/Bound/BoundNodes.cs` — `BoundNode`/`BoundExpression`/`BoundInvariant` +
  the VO-invariant-reachable forms only (Binary/Unary/Reference/Literal/MemberAccess/Call + Conditional/
  Coalesce/Match/Guard/Let; Lambda only if reachable).
- New `src/Koine.Compiler/Ast/Bound/Lowerer.cs` — `KoineSyntaxVisitor<BoundNode>`; hosts relocated
  `SourceText`/`SourceTextVisitor` (Koine syntax only).
- New `src/Koine.Compiler/Ast/Bound/BoundModel.cs` — `InvariantsFor(vo)`; `ValueObjectDecl`-keyed cache.
- Modify `src/Koine.Compiler/Ast/SemanticModel.cs` — lazy `_bound`; internal
  `BoundInvariantsFor(ValueObjectDecl)`.
- Modify `src/Koine.Compiler/Emit/CSharp/CSharpEmitter.cs` + `CSharpEmitter.ValueObjects.cs` —
  **additive** `BoundInvariant` overload for the VO path; `SourceText` reads from `Ast/Bound/`.
- New `tests/Koine.Compiler.Tests/BoundIrLoweringTests.cs`.
- **`Emit/TypeScript/`, `ModelIndex.cs`, `TypeResolver.cs`, `KoineType.cs` unchanged.**

**Verification gate (all must be green; final phase).**
- `./build.sh` green (~500 tests) — un-migrated emitter paths + TS emitter unaffected.
- **Byte-identical snapshots:** full Verify suite (`Snapshots/`, `Conformance/Snapshots/`) green with
  **ZERO `.received.txt` diffs** (the slice's hard contract).
- **Roslyn meta-test:** emitted VOs still throw `DomainInvariantViolationException` with identical
  `rule:`/message text.
- `BoundIrLoweringTests`: (1) resolution — every `BoundInvariant.Condition.Type` non-null; every
  `BoundReference.Symbol` `Assert.Same` as `GetSymbolInfo(node)`; every `Syntax` is its origin;
  (2) message default — explicit preserved, missing == relocated `SourceText`; (3) reference-key identity
  + `ValueObjectDecl` cache uses ref-comparer (two same-named VOs across contexts don't collide);
  (4) coverage/no-throw — hand-built model of every reachable form lowers; unknown name →
  `BoundReference(ErrorSymbol)`.

**Rollback note.** The new `Ast/Bound/` files are additive; the only emitter change is one additive
overload + the `SourceText` relocation. Revert the phase commit to restore the inline VO message default
and remove the bound layer; entity/command paths were never touched. Terminal phase — reverting it
unwinds nothing else.

## Open questions for the human (decide before implementation)

1. **Source-generated vs hand-written visitor (Commit 2).** Plan assumes a **Roslyn incremental source
   generator** (drift-proof, mirrors Roslyn-from-`Syntax.xml`) at the cost of a new `netstandard2.0`
   analyzer project + analyzer-host build complexity. Acceptable, or prefer a hand-written visitor with a
   reflection "every-node-handled" test (smaller build surface, more drift risk)? **This is the biggest
   architectural fork.**
2. **Symbol-identity model (Commit 3).** Plan **extends the existing `abstract class Symbol`** (interning
   + `init` `ContainingSymbol`, zero caller churn) rather than a parallel `ISymbol`. Confirm — and confirm
   the **`ContextSymbol`** name (not `NamespaceSymbol`) and the new `Symbol` subclasses' visibility
   (internal binder internals; public `GetSymbolInfo`/`GetDeclaredSymbol` on `SemanticModel`).
3. **Cross-document identity scope (Commit 3).** Plan ships **in-document identity only**; cross-file
   stays `DeclSpan`-value. Confirm that a workspace-level binder is explicitly **out of scope** for this
   effort.
4. **Which emitter slice Commit 4 retargets.** Plan chooses **value-object invariant guards** (exercises
   resolution + a real desugaring, self-contained, snapshot-bounded). Alternative first slices considered
   and rejected: VO field projection (less expression resolution), whole `EmitValueObject` (too broad),
   any entity/command/CQRS path (max blast radius). Confirm the VO-invariant slice.
5. **`SourceText`/`SourceTextVisitor` relocation (Commit 4).** Plan **moves** them into `Ast/Bound/`
   (proving the layer absorbs the desugaring) vs leaving them in `CSharpEmitter` and merely calling them
   (smaller diff, desugaring not truly absorbed). Plan moves — confirm, since it touches a shared emitter
   helper.
6. **`BCL ReferenceEqualityComparer` vs a promoted shared `Ast/NodeIdentityComparer`.** Plan uses the BCL
   comparer in Commits 3/4 to avoid touching the Commit-1 `SyntaxGraph.cs`. If a shared `Ast/` comparer is
   preferred for consistency, that promotion (and the `SyntaxGraph.cs` edit) is a small add — decide.
7. **Generator test project (Commit 2).** Plan relies on end-to-end exercise via `Koine.Compiler.Tests`
   (the oracle proves correctness) and **defers** a dedicated `Koine.Compiler.SourceGen.Tests`
   generator-snapshot project. Acceptable, or require it now?
